## TypeScript 란
Typescript는 2012년 10월에 Microsoft에서 public 되었습니다. <br/>
또한 아래의 3가지를 중심으로 디자인되었습니다.

1. strongly typed
2. object oriented
3. compiled language

## TypeScript의 특징
<div style="color: #FE8B44;margin-bottom:12px;">TypeScript is JavaScript for application-scale development</div>
JavaScript의 장점과 추가적인 기능을가지고 있습니다.

1. Compilation
> js는 인터프린터 언어입니다. <br/>
직접 테스트를 해보지 않으면에러 및 버그를 발견의 어려움을 겪는 경우가 있습니다. <br/>
TypeScript의 경우에는 코드를 컴파일하는 과정에서 에러를 발견 할 수 있습니다.
2. Strong Static Typing
> js는 타입에 있어서 자유롭습니다. <br/>
> ts는 TLS(TypeScript Language Service)를 통한 <br/>
> 선택적 정적 입력 및 유형 추론 시스템과 함께 제공됩니다. <br/>
> (또한 새로운 타입을 정의할 수 있습니다.)
3. Object Oriented Programming
> js를 사용하여 OOP 구현하려고 하면 prototype과 constructor
functions로 구현가능 <span style="color: #FE8B44;font-weight:bold;">BUT</span> 불편 <br/>
> (ES6넘어오면서 class를 사용 할 수 있으나 결국 prototype베이스) <br/>
> ts를 사용하게 되면 <span style="color: #FE8B44;font-weight:bold;">class, interface, generics, types</span>
를 사용하게되어 강력한 OOP를 구현할 수 있습니다.

## 공부해야하는 이유
아래의 내용은 주관적인 내용이 포함되어있어 참고해주시기를 바랍니다.
1. Document
> js를 사용하는 개발자라고 하면 죄다 ts로된 document를 읽는게 속도가 정말 빠릅니다.
2. Type Safe
> js로 UT를 작성하게되면 큰 걱정이없지만 그 만큼 테스트케이스도 늘어납니다. <br/>
> 타입에대한 error를 테스트를 작성하면... <br/>
> 컴파일에서 에러 발생하는것을 아주 감사하게 여깁니다 (당해본 자)
3. accessibility
> js에대한 이해도가 있다면 습득 속도가 빠르다고 생각됩니다. <br/>
> TypeScript is JavaScript for application-scale development
4. class사용이 즐겁다.
> js class는 private/public 이용도 getter/setter 등이 전부 <br/>
> 구현을 통하여 극복을 했어야하는데 편합니다 즐겁습니다.
