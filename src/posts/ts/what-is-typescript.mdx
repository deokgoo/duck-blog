## <div style="color: #FE8B44">TypeScript 란</div>
Typescript는 2012년 10월에 Microsoft에서 public 되었습니다. <br/>
또한 아래의 3가지를 중심으로 디자인되었습니다.

1. strongly typed
2. object oriented
3. compiled language

## <div style="color: #FE8B44">TypeScript의 특징</div>
<div style="color: #FE8B44">TypeScript is JavaScript for application-scale development</div>
JavaScript의 장점과 추가적인 기능을가지고 있습니다.

1. Compilation
> JavaScript는 인터프린터 언어이다. <br/>
직접 테스트를 해보지 않으면에러 및 버그를 발견의 어려움을 겪는 경우가 있다. <br/>
TypeScript의 경우에는 코드를 컴파일하는 과정에서 에러를 발견 할 수 있습니다.
2. Strong Static Typing
> JavaScript는 타입에 있어서 자유롭습니다. <br/>
> TypeScript는 TLS(TypeScript Language Service)를 통한 <br/>
> 선택적 정적 입력 및 유형 추론 시스템과 함께 제공됩니다.
3. type definitions
4. Object Oriented Programming

superset of javascript
dynamically typed => statically typed
js는 dynamically typed이기 때문에 컴파일시에 에러를 잡지를 못하게 되어
런타임에서 발견하게되어 불편을 겪는 경우가 발생한다. 허나 ts를 사용하게 되면
statically typed이기때문에 컴파일 단계에서 타입 에러를 발견 할 수 있다.

또한 객체지향 프로그래밍을 구현하려고 하면 기존 js는 prototype과 constructor
functions로 객체 지향을 구현할 수 있지만 아무래도 불편한 부분이 있다.
(ES6넘어오면서 class를 사용 할 수 있으나 결국 prototype베이스 이기때문에 버겁다)

ts를 사용하게 되면 class, interface, generics, types를 사용하게되어
강력한 객체지향 프로그래밍을 사용할 수 있게 됩니다.
